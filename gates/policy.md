# 기계 게이트 정책

원칙: **게이트는 사람이 아니라 기계다.** 너(PI)는 매번 승인하는 공증인이 아니라, 막힐 때만 보는 **항소심**.

## permissionless 내부 (사람 게이트 없음)
가설 생성, 에이전트/팀 구성, 샌드박스 실험, 내부 노트/위키, research 브랜치 코드, 자기비평, dead-end 기록 — 전부 자유. (가역적·내부)

## 기계 게이트 perimeter (되돌릴 수 없는 경계만)
| 경계 | 게이트 (기계) |
|---|---|
| 돈 / 컴퓨트 지출 | 예산 캡 (런당 · 일일) |
| 결과 / 주장 공개 | 적대적 비평 정족수 통과 (과정·실패는 자유) |
| Elnath main 머지 | 그린 테스트 + 비평 통과 |

## 하드 스톱 (항상 금지)
법적 행위, 타인 계정 / 타인 돈, 사칭. 자격증명·결제 없음 / repo 밖 파괴 시 정지.

## 인식적 지위로 가른다
- "우리가 X를 시도했고 망했다(미검증)" = **자유 공개**.
- "X가 된다 / Y를 이겼다(주장)" = **비평 정족수 필요**.
- (Elnath의 claim boundary 규율 계승)

## 신뢰 사다리
게이트는 처음엔 켜둔다(비평 레이어 미검증). 비평이 *실제로 slop을 거른다는 증거*가 쌓이면 하나씩 자동화 → permissionless로 졸업. **완전 permissionless는 증거로 획득한다.**
