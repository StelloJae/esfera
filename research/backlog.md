# 리서치 백로그 (아젠다)

## 주제 선정 규칙 — portfolio
매 라운드 주제는 비율로 뽑는다 (다양성을 *시스템*으로):
- **2 grounded** — Elnath dogfood에서 실제로 깨진 것 / 실제 사용자 니즈에서.
- **1 wildcard** — 엉뚱하거나 바보같아 보이는 시도. (Thinking Diversity 슬롯)

가짜병렬(에이전트 난사로 metric 채우기) 금지 — 유망한 쪽으로 모이고 막히면 재조직.

> 아래 grounded 후보는 Elnath 현 상황 문서(`elnath-current-state-2026-06-13`)에서 실제로 깨졌거나 미완인 곳을 *연구 질문*으로 바꾼 것. 버그 티켓이 아니라, "믿을 수 있는 자율 에이전트"의 열린 문제로 다룬다.

## Grounded 후보 (실제 Elnath dogfood)
- [ ] **불안정한 외부 의존성 위의 신뢰성** — 무인 캘린더 프리페치가 "맥 깨어있음 + Chrome 로그인" 전제라 깨지기 쉬움. → 자율 에이전트가 *깨지기 쉬운 외부 의존성* 위에서 정직하게 degrade 하며 신뢰성 있게 도는 법?
- [ ] **교정 기억 (S4)** — 사용자가 "날씨 빼" 하면 다음 브리핑이 반영해야 함(능력 사다리 L4, 미완). → 자율 에이전트가 사용자 교정을 기억하고 다음 행동에 반영하는 *최소* 메커니즘?
- [ ] **승인 대기 → 재개 (S5)** — 에이전트가 승인 받으려 멈췄다가 나중에 *상태 손실 없이* 이어서 재개(L4, 미완). → 우리 "항소심" 모델의 핵심: PI가 아침에 승인하면 밤일이 정확히 이어지는 법?
- [ ] **침묵 실패의 표면화** — OpenAI provider가 이미지 블록을 *조용히* 버림. → 에이전트의 *조용한 기능 구멍*을 사용자에게 정직하게 드러내는 법? (provider 없으면 정직 에러 내는 기존 원칙의 일반화)
- [ ] **능력 경계의 정직한 표면** — `elnath briefing daily`가 AI 브리핑이 아닌데 비개발자가 헷갈림. → 비개발자에게 에이전트가 *무엇을 할 수 있고 없는지*를 오해 없이 드러내는 네이밍/UI 원칙?
- [ ] **한 인격의 환상** — 내부 멀티에이전트를 하나의 파트너로 요약(one-being illusion). → 시끄럽지 않은 *하나의 파트너*로 표면화하는 법? (Esfera 자신의 swarm에도 적용)

## Wildcard 후보
- [ ] 에이전트가 "오늘은 아무것도 안 하는 게 최선"이라 판단하고 *침묵*하는 능력? (저소음 능동성)
- [ ] 에이전트가 자기 실패를 스스로 graveyard에 묻고 "이 길은 가봤다"를 사용자에게 보여주면 신뢰가 오르나?

## 큰 열린 질문 (북극성 근처)
- 비개발자가 자율 에이전트를 *신뢰*하게 만드는 조건은?
- 인간 승인이 아니라 *가역성(reversibility)*으로 자율성을 안전하게 만드는 법? ← Esfera 대표 연구주제
